PEFPAR GeoNode / About / API

The PEFPAR GeoNode is a digital service provided by the U.S. Department of State, Humanitarian Information Unit (HIU) to support interagency collaboration for the development and maintenance of PEPFAR's spatial data.

Users / GIS Analysts

Users and GIS analysts can connect to the PEPFAR GeoNode via QGIS, ArcMap, and TileMill. In QGIS, users can add PEPFAR GeoNode layers as WMS or WFS layers. See the QGIS page in the documentation for more detail. In ArcMap, use the interoperability extension to add a WMS layer to ArcCatalog. All the layers should then be available. In TileMill, simply grab the download url from each respective layer page and use it as the path.

Developers

Developers can interact with the PEPFAR GeoNode through 3 APIs: CSW, OGC Services, and the internal GeoNode API.

PEPFAR GeoNode uses PyCSW to produce standardized CSW metadata services.

The CSW endpoint is: /catalogue/csw. To retrieve metadata records run something like this: /catalogue/csw?request=GetRecords&service=CSW&version=2.0.2&resultType=results&maxRecords=1000&elementsetname=full&typenames=csw:Record&outputschema=http://www.isotc211.org/2005/gmd. This endpoint can be used by other systems to automatically harvest records from the PEPFAR GeoNode.

All layers in the PEPFAR GeoNode are accessible via OGC Services. The WMS Capabilities document is accessible with: /geoserver/wms?SERVICE=WMS&REQUEST=GetCapabilities. This endpoint can be used by geospatial analysts to pull layers directly into their desktop environment (/docs/integration.html). OGC services can also be used to pull layers directly from the PEPFAR GeoNode into web maps to make mashups.
